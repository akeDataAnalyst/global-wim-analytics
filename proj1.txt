Project 1: Global Inventory Analytics Dashboard

Description: An interactive web application built with Python (using libraries like Pandas, NumPy, and Plotly) and deployed via Streamlit, simulating a global warehousing and inventory management (WIM) system for humanitarian operations. 
The dashboard allows users to upload inventory datasets (e.g., CSV files from multiple country offices), perform real-time analytics, visualize stock levels, trends, and KPIs, and generate compliance reports. 
Features include data cleaning, forecasting (using simple time-series models), and customizable filters for multi-country views.

Problem and Solution: 
Problem: In decentralized humanitarian supply chains like CARE's (spanning 60+ warehouses across 50+ countries), tracking inventory accuracy, identifying stock discrepancies, and ensuring compliance with donor requirements can be challenging due to fragmented data from diverse locations, leading to inefficiencies, audit risks, and delayed aid delivery.
Solution: The dashboard addresses this by integrating mock or sample datasets (e.g., from open humanitarian data sources like HDX) to demonstrate data migration, quality checks, and visualization. Python scripts handle backend processing (e.g., aggregating data across "country offices," calculating KPIs like stock turnover rates or expiry risks), while Streamlit provides a user-friendly interface for non-technical users to run simulations, test scenarios (e.g., emergency stock pre-positioning), and export audit-ready reports. This mirrors CARE's WIM system testing, UAT, and analytics responsibilities.

Recommendation: This project is ideal for showcasing your fit for the role as it directly demonstrates experience in data analytics/visualization, WIM system support (e.g., mimicking ERP data flows), and global compliance tools—key required qualifications. Host it on GitHub with a live Streamlit demo (via Streamlit Sharing) and include a README explaining how it could scale to CARE's operations, emphasizing humanitarian contexts like emergency response in Ethiopia or Africa. 
It highlights your ability to build capacity-building resources (e.g., embedded user guides) and contribute to global KPIs/dashboards.
